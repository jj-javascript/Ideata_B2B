## Findings (machine-oriented format)

```
// 🔴 [A11Y] WCAG 2.1.1 / 4.1.2: Board row actions are off-screen until hover and the toolbar is wrapped in aria-hidden, while focusable buttons remain inside—keyboard and screen-reader users cannot reliably discover or use rename, priority, share, or delete.
//    Where: components/SwipeBoardCard.tsx (gradient overlay div with aria-hidden; buttons translated off-canvas until .group-hover)
//    Fix: Remove aria-hidden from the interactive toolbar. Provide a keyboard-accessible disclosure pattern (e.g. visible “Board actions” menu button with aria-expanded, or always-visible icon buttons with aria-label). Ensure focus moves to revealed controls and that tab order is not hidden behind hover-only CSS.

// 🔴 [A11Y] WCAG 2.1.1 / 4.1.2 / 2.4.3: Modal overlays are plain positioned divs without dialog semantics, focus trap, Escape-to-close, or focus return to the triggering control.
//    Where: app/dashboard/page.tsx (new board, rename, priority, share, delete, delete-all confirmations); app/meeting/[id]/page.tsx (invite modal)
//    Fix: Use role="dialog", aria-modal="true", aria-labelledby pointing at the modal heading id; implement Escape handler; trap Tab within the modal while open; on close, restore focus to the element that opened the dialog. Prefer <dialog> with showModal() or a vetted headless dialog primitive.

// 🔴 [A11Y] WCAG 1.3.1 / 3.3.2: Several inputs are not programmatically labeled; validation/error text is not tied to fields with aria-describedby / aria-invalid.
//    Where: app/dashboard/page.tsx (new board title input—placeholder only; rename input; share email uses sibling <p> not <label htmlFor>); app/meeting/[id]/page.tsx (invite email input); app/board/[id]/page.tsx (inline title edit input); components/MeetingScheduler.tsx (error paragraph vs fields)
//    Fix: For each control, add <label htmlFor="unique-id"> matching input id. For errors: set aria-invalid={true} when invalid and id on error element; reference it from the input with aria-describedby="err-id".

// 🟡 [A11Y] WCAG 2.4.4: Navbar “Pricing” navigates to /#pricing but no id="pricing" (or equivalent) exists on the marketing home page.
//    Where: components/Navbar.tsx; app/page.tsx
//    Fix: Add a section with id="pricing" and substantive content, or remove/replace the nav item with a real destination (e.g. /pricing or an in-page anchor that exists).

// 🟡 [A11Y] WCAG 2.4.2: Only the root layout exports metadata.title; primary app routes are client components and do not define unique document titles.
//    Where: app/layout.tsx vs app/dashboard/page.tsx, app/board/[id]/page.tsx, app/meeting/[id]/page.tsx, etc.
//    Fix: Add small server layout.tsx files per segment with export const metadata = { title: "… | Ideata" }, or call document.title in client pages inside useEffect when data loads (less ideal for SSR but better than a single global title).

// 🟡 [A11Y] WCAG 4.1.2: Icon-only and pseudo-button controls lack dependable accessible names; decorative SVGs are not consistently hidden from the accessibility tree.
//    Where: components/AiIdeaInput.tsx (mic button uses title only); components/SwipeBoardCard.tsx (icon buttons use title only); components/FloatingVideoPanel.tsx (minimize/close: title only; decorative SVGs); components/VideoRoom.tsx (retry control)
//    Fix: Add aria-label on each icon-only button mirroring the visible intent. Mark decorative SVGs with aria-hidden="true" (and focusable="false" where applicable) when the name comes from the button.

// 🟡 [A11Y] WCAG 2.1.1: Minimized floating video chip uses role="button" but only Enter activates; Space should also work, and the control needs an explicit accessible name.
//    Where: components/FloatingVideoPanel.tsx
//    Fix: Prefer a real <button type="button"> for the minimized chip. If keeping role="button", handle both Enter and Space (preventDefault on Space) and set aria-label="Restore video panel" (or similar).

// 🟡 [A11Y] WCAG 1.3.1: “Video platform” in the scheduler is a bare <label> with no htmlFor; the three options are separate buttons without a grouped field pattern.
//    Where: components/MeetingScheduler.tsx
//    Fix: Wrap the three platform buttons in a <fieldset> with <legend>Video platform</legend>, or use role="radiogroup" with aria-labelledby on a visible heading id.

// 🟡 [A11Y] WCAG 3.3.1 / 4.1.3: Share failure uses window.alert, which is disruptive and separates errors from the form context; async loading and “processing” states are mostly visual only.
//    Where: app/dashboard/page.tsx (handleAddImage catch uses alert); components/AiIdeaInput.tsx (processing state); various “Loading…” strings
//    Fix: Inline errors near the share form with aria-live="polite" on a status region; on the mic button use aria-busy={status === "processing"} and/or an adjacent aria-live region for status changes.

// 🔵 [A11Y] WCAG 2.4.1: No skip link to bypass repeated navigation to main content.
//    Where: app/layout.tsx
//    Fix: First focusable element: visually hidden skip link that becomes visible on focus, href="#main-content", and ensure each page’s primary <main> has id="main-content".

// 🔵 [A11Y] WCAG 2.4.7: Several fields use outline-none with only a border color change on focus; contrast/salience of the focus indicator may be weaker than a dedicated focus ring.
//    Where: app/dashboard/page.tsx, app/meeting/[id]/page.tsx, app/board/[id]/page.tsx
//    Fix: Add focus-visible:ring-2 focus-visible:ring-brown-600 focus-visible:ring-offset-2 (or equivalent) in addition to border changes, matching patterns already used in components/MeetingScheduler.tsx.

// 🔵 [A11Y] WCAG 2.1.1: Video panel resize affordance is mouse-only (onMouseDown).
//    Where: components/FloatingVideoPanel.tsx
//    Fix: Provide keyboard-resize alternatives or document resize in settings; at minimum expose resize via visible controls reachable by keyboard.

// 🔵 [A11Y] Best practice: Some <button> elements omit type="button" (implicit submit behavior can surprise users when markup changes).
//    Where: app/meeting/[id]/page.tsx (“Join with video”, invite “Close”); components/VideoRoom.tsx (“Retry”)
//    Fix: Add type="button" explicitly on non-submit buttons.

// 🔵 [A11Y] Best practice: Collaborator avatars use title on a wrapping div; prefer aria-label on the container for more consistent exposure than title alone.
//    Where: app/board/[id]/page.tsx
//    Fix: aria-label={`Collaborator: ${c.userName}`} on the avatar wrapper (and keep alt text on images as today).

---

## Positive notes

- `app/layout.tsx` sets `lang="en"` on `<html>`.
- Landmarks are used in several places (`header`, `main`, `nav`, `footer` on the landing page).
- `components/MeetingScheduler.tsx` uses proper `<label htmlFor>` for most fields and stronger focus rings than other forms.
- External links (e.g. LiveKit help link in `VideoRoom`) use `rel="noopener noreferrer"`.
